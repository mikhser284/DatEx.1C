1.		Первичная синхронизация объектов из 1С в Creatio
—————————————————————————————————————————————————————————————————————————————————————————————————————————————————————————————————————————————————————


1.1.	ОПИСАНИЕ РЕШЕНИЯ
В даном решении создано для демонстрации того как может проходить первичная синхронизация данных между объектами из 1С и Creatio (т.е. при первом запуске Creatio)

Как именно происходит проецирование полей объектов из 1С на поля объектов в Creatio в большей степени отображено при помощи атрибутов объектов из объектных моделей 1С и Creatio,
а также атрибутов примечаний. Так же следует отменить что важное значение на то как синхронизируються данные между системами влияет объект настроек синхронизации.

1.2.	ПОЧЕМУ НЕ БЫЛО ВОЗМОЖНО НАПРЯМУЮ ВОССОЗДАТЬ ПРОЕЦИРОВАНИЕ СВОЙСТВ ОБЪЕКТОВ ИЗ 1С НА СВОЙСТВА ОБЪЕКТОВ В CREATIO (В EXCELL)
В некоторых случаях (для некоторых полей) не возможно настроить синхронизацию таким образом чтобы напрямую проециоровать свойства объекта из 1С на свойство объекта из Creatio. 
Нужны были промежуточные объекты и промежуточная логика работы механизма синхронизации, а так же объект настроек синхронизации.

1.3.	СТРУКТУРА РЕШЕНИЯ
• В решении воссозданы модели данных объектов получаемых из 1С (DatEx.OneC.DataModel) и в последствии объектов записываемых в Creatio (DatEx.Creatio.DataModel).
• В некоторых случаях, для удобства, объекты дополнены промежуточными свойствами.
• Вся работа по переносу данных осуществляется в проекте "App" в методе Programm.Main
